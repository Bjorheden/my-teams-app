# Backend – MyTeams FastAPI

## Overview

Python 3.12 + FastAPI backend. Mock data only – no external APIs.

## Project layout

```
backend/
├── Dockerfile
├── pyproject.toml
└── app/
    ├── main.py          ← app factory, mounts routers
    ├── api/
    │   ├── health.py    ← GET /v1/healthz, GET /v1/readyz
    │   ├── teams.py     ← CP2: team search
    │   └── me.py        ← CP2: follows + dashboard
    ├── core/
    │   └── config.py    ← settings from env vars
    ├── mockdata/
    │   └── fixtures.py  ← hardcoded teams, fixtures, standings
    └── tests/
        └── test_health.py
```

## Running locally (without Docker)

```bash
cd backend

# Create and activate a virtual environment
python -m venv .venv
source .venv/bin/activate          # Linux/macOS
# .venv\Scripts\Activate.ps1       # Windows PowerShell

# Install runtime + dev dependencies
pip install -e ".[dev]"

# Start the server
uvicorn app.main:app --reload --port 8000
```

## Running with Docker (preferred)

```bash
# From repo root:
make up          # build + start
make logs        # tail logs
make down        # stop
```

## API Endpoints (Checkpoint 1)

| Method | Path | Description |
|--------|------|-------------|
| GET | `/v1/healthz` | Liveness probe – is the process alive? |
| GET | `/v1/readyz` | Readiness probe – is the app ready to serve? |
| GET | `/docs` | Swagger UI (auto-generated by FastAPI) |
| GET | `/redoc` | ReDoc UI |

## Running tests

```bash
# Via Docker (recommended)
make test

# Locally
cd backend
pytest app/tests -v
```
